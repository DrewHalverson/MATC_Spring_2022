Linking Bootstrap

Don't forget the meta. Be sure to have the two minimum meta tags for the charset and the viewport defined.

Versions matter. When linking Bootstrap, be sure to have the associated Bootstrap CSS, Popper, and JavaScript links. Don't mix the links between different versions of Bootstrap! While you will have to have the Bootstrap CSS, you may not need the Popper or JavaScript links. It depends on what aspects of Bootstrap you are planning on using. Remember: Bootstrap v3(+) uses float for layout, but Bootstrap v4(+) uses flexbox. We will be using Bootstrap 5 which began with Bootstrap v4.6.

Customize styling. Unless you are planning on using bootstrap default stylings, you will need to link your custom CSS. Be sure that your custom CSS comes AFTER Bootstrap links.

Container Decisions

Containers are the most basic layout element in Bootstrap and is required when using the Bootstrap grid system.

Although containers can be nested, most layouts do NOT require a nested container, and it should be avoided when possible.

  • Fixed width grid layout container. class="container" A responsive, fixed-width container where the max-width changes at each pre-defined breakpoint.
  • Blended width. class="container-{breakpoint}"A mix of full width and fixed width. The container will stay full width until the breakpoint and then transition to a container after the breakpoint indicator.
  • Full width layout. class="container-fluid" Sometimes referred to as fluid-width, full width has a width of 100% all of the time.

Responsive Breakpoints

Mobile First. Bootstrap is mobile first, so media queries have the smallest breakpoint as the default. Breakpoints are mainly based on minimum viewport widths and scale up as the viewport changes.

Breakpoints by versions. Bootstrap v3(+) has different breakpoints (aka media query ranges) from Bootstrap v4(+) and v5(+). It is important to understand what those breakpoints are for the version you are applying.

Predefined Breakpoints Bootstrap has predefined grid breakpoints. Bootstrap v3(+) has four breakpoints (xs (default), sm, md, lg) whereas Bootstrap v4(+) has five breakpoints (xs (default), sm, md, lg, xl). With Bootstrap v4.6(+) and Boostrap v5(+), there are now six breakpoints (default, sm, md, lg, xl, xxl)

Grid System

12 Column System The grid system is based off 12 column system and responsive tiers. The grid system uses a series of containers, rows, and columns to layout and align content. Do not exceed 12 columns per row.

Wrapper Element Relationships The container, row, and columns have very specific relationships.

  • The container is the wrapper parent element to the row (child element).
  • The row is the wrapper parent element to the column(s).
  • Columns can only be immediate children of rows.
  • You must have all three components. Do not skip levels!
